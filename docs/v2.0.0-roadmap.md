# context-mem v2.0.0 Roadmap — "The Intelligence Layer"

> v1.0 = memory (remember + search). v2.0 = intelligence (reason + connect + predict).

**Created:** 2026-03-27
**Target:** v2.0.0

---

## Vision

Transform context-mem from a "memory tool" into an "intelligence layer" — not just storing and retrieving, but understanding relationships, transferring knowledge across projects, and proactively surfacing relevant context.

---

## Tier 1 — Game Changers

### 1. Cross-Project Knowledge Transfer
- [x] Global knowledge store at `~/.context-mem/global/`
- [x] Auto-promote frequently used patterns from project → global
- [x] "Learn once, use everywhere" — PostgreSQL patterns, error fixes, API conventions
- [x] Privacy controls — opt-in per project, never auto-share secrets
- [x] MCP tool: `global_search(query)` — search across all projects
- [x] MCP tool: `promote_knowledge(id)` — promote entry to global store
- [x] Merge/conflict resolution for duplicate entries across projects

**Impact:** Huge | **Effort:** Medium | **ROI:** Very High

### 2. Proactive Context Injection
- [x] PostToolUse hook analyzes what Claude is doing (file reads, edits)
- [x] Auto-inject relevant knowledge/observations without Claude asking
- [x] "Last time you edited auth.ts, the issue was expired JWT refresh tokens..."
- [x] "This module has 3 known edge cases: [list]"
- [x] Configurable injection threshold (relevance score cutoff)
- [x] Rate limiting — max N injections per minute to avoid noise
- [x] SessionStart hook injects project-specific "briefing"

**Impact:** Huge | **Effort:** Medium | **ROI:** Very High

### 3. Knowledge Graph
- [x] Entity-relationship model replacing flat knowledge entries
- [x] Relationships: `uses`, `depends-on`, `fixed-by`, `contradicts`, `supersedes`
- [x] Entity types: file, module, pattern, decision, bug, person, library
- [x] Graph queries: "What depends on auth module?" → traversal
- [x] Migration from flat knowledge → graph (backwards compatible)
- [x] MCP tool: `graph_query(entity, relationship_type, depth)`
- [x] MCP tool: `add_relationship(from, to, type)`
- [x] Dashboard: graph visualization (D3.js or similar)

**Impact:** Huge | **Effort:** Large | **ROI:** High

### 4. Multi-Agent Shared Memory
- [x] Multiple Claude sessions share same knowledge base
- [x] Task coordination — "Agent A is working on auth, Agent B on payments"
- [x] Conflict resolution — concurrent knowledge writes
- [x] Shared event stream for cross-agent awareness
- [x] Lock-free architecture (SQLite WAL already supports this)
- [x] MCP tool: `agent_status()` — see what other agents are doing
- [x] MCP tool: `claim_task(task_id)` — prevent duplicate work

**Impact:** High | **Effort:** Large | **ROI:** Medium

## Tier 2 — Power Features

### 5. Time-Travel Debugging
- [x] "Show project state 3 days ago" — snapshot diffs
- [x] Knowledge delta — what was added/changed/removed between dates
- [x] Observation timeline replay with filtering
- [x] MCP tool: `time_travel(date, scope)` — restore context from any point
- [x] Dashboard: interactive timeline explorer

**Impact:** Medium | **Effort:** Medium | **ROI:** Medium

### 6. Natural Language Query Engine
- [x] "What did we decide about authentication?" — intent understanding
- [x] Optional local LLM integration (Ollama/llama.cpp)
- [x] Falls back to existing 4-layer search when LLM unavailable
- [x] Query decomposition — complex questions → multiple searches
- [x] MCP tool: `ask(question)` — natural language over knowledge

**Impact:** High | **Effort:** Medium | **ROI:** High

### 7. Custom Summarizer Plugin API
- [x] Community-created summarizers as npm packages
- [x] `context-mem plugin add @community/k8s-summarizer`
- [x] Plugin interface: `detect(content) → boolean`, `summarize(content) → string`
- [x] Registry of community plugins
- [x] Hot-reload without server restart

**Impact:** High | **Effort:** Medium | **ROI:** High

### 8. Streaming Observations
- [x] Real-time observation stream via WebSocket
- [x] Dashboard live feed (currently HTTP polling)
- [x] Event-driven architecture for plugins
- [x] Server-Sent Events (SSE) as lightweight alternative

**Impact:** Medium | **Effort:** Small | **ROI:** Medium
